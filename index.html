什么是 LLM Wiki？

LLM Wiki 是一种新型的个人知识管理方法。传统Wiki需要你手动编辑和维护，而LLM Wiki由AI助手自动维护。

传统Wiki 😓
  • 每篇文章手动编辑（1-2小时）
  • 格式不一致
  • 忘记添加链接
  • 索引更新不及时
LLM Wiki 🚀
  • AI自动编译（5-10分钟）
  • 格式统一标准
  • 自动关联概念
  • 索引实时更新
🤖
AI自动维护

添加素材后，AI自动提取概念、生成概念页、建立关联网络。

🔗
双向链接

概念之间自动建立链接，形成知识网络，轻松发现关联。

💬
自然语言查询

直接用中文提问："解释一下自由能原理"，AI会基于你的知识库回答。

🔒
本地存储

所有数据存储在本地，不上传云端，完全由你控制，保护隐私。

5分钟快速开始

安装Obsidian

从 obsidian.md 下载并安装Obsidian（免费）。

初始化项目

运行初始化脚本，创建标准目录结构：

# 创建项目目录
mkdir ~/my-llm-wiki

# 初始化
python3 ~/.qclaw/skills/llm-wiki-starter/scripts/init_llm_wiki.py ~/my-llm-wiki
用Obsidian打开

打开Obsidian → "Open folder as vault" → 选择 ~/my-llm-wiki

添加第一个素材

把一篇文章保存到 raw/articles/ 目录：

# 保存文章
echo "# 自由能原理\n内容..." > ~/my-llm-wiki/raw/articles/2026-04-30-free-energy.md
编译知识

让AI帮你编译素材：

python3 ~/.qclaw/skills/llm-wiki-starter/scripts/compile_raw.py ~/my-llm-wiki

或在OpenClaw中说："请编译raw目录的素材"

常见问题

Q：我需要懂编程吗？
A：完全不需要！ LLM Wiki专为非技术用户设计。你只需要会基本的电脑操作（复制粘贴文件），会用自然语言与AI对话即可。
Q：数据安全吗？
A：非常安全。所有数据存储在你的本地电脑，不上传到云端。AI只在你请求时访问你的文件，完全由你控制。建议定期备份到云盘或外部硬盘。
Q：为什么要用Obsidian？
A：Obsidian是最佳的双向链接笔记工具。它支持：
• 双向链接 [[概念名]]
• 图谱可视化
• Markdown原生支持
• 本地存储
• 免费使用
也可以用Logseq等类似工具替代。
Q：编译后的概念页质量如何保证？
A：AI生成 + 人工审核 = 最佳实践。
1. AI生成初稿（5分钟）
2. 快速检查（2分钟）- 链接、内容、格式
3. 深度完善（按需）- 补充理解、应用案例、概念网络
建议小批量编译，及时检查。
Q：多久编译一次比较合适？
A：推荐"少量多次"。
• 日常：每天1-2篇
• 周末：批量处理本周素材
• 避免：积压几十篇一次性编译（质量难保证）
最高效的方式：看到好文章立即编译。
